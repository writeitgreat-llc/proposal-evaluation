# proposal-evaluation -- parameterised routes that are intentionally reachable
# without a login, because the URL itself is the credential.
#
# Format:  <url rule>   # why it is safe
#
# Adding a line here is a security decision. Before you add one, answer:
#   1. Is the parameter unguessable? (>= 128 bits of randomness, not sequential)
#   2. Is the data behind it disclosable to anyone holding the link?
#   3. Does the link expire?
# If the answer to any of those is no, add a guard instead of an entry here.

# --- password reset: uuid4 hex (122 bits), single use, 1 hour expiry ---
# Author.generate_reset_token / AdminUser / Publisher all use uuid.uuid4().hex
# and set password_reset_expires = utcnow() + 1h; verify_reset_token() rejects
# expired tokens and the handler clears the token after a successful reset.
/author/reset-password/<token>       # uuid4 reset token, 1h expiry, cleared on use
/publisher/reset-password/<token>    # uuid4 reset token, 1h expiry, cleared on use
/admin/reset-password/<token>        # uuid4 reset token, 1h expiry, cleared on use

# --- evaluation results: capability URL mailed to the author ---
# FIXED 2026-07-21 (security/results-tokens). The old note here warned that
# generate_submission_id() ("WIG-<YYYYMMDD>-<5 hex>", ~20 bits behind a
# guessable date) was the only key to these pages. Now every NEW proposal gets
# results_token = secrets.token_urlsafe(16) (128 bits) and these routes resolve
# through _resolve_proposal_results_key(): token first, and a token-bearing row
# addressed by its short submission_id 404s. check_route_guards.py therefore
# classifies them GUARDED (resolver helper), so these entries are dormant.
# They stay listed deliberately: LEGACY rows (results_token IS NULL, created
# before the migration -- there is intentionally no backfill) are still served
# by their low-entropy submission_id so links already emailed to authors keep
# working. That residual surface shrinks to zero as old rows age out; if it
# ever needs to be closed early, backfill tokens and re-send the links --
# do NOT add new low-entropy keys on this pattern.
/results/<submission_id>             # token-first; submission_id honoured only for legacy token-less rows
/download/<submission_id>            # PDF of the same report, same rule
/api/status/<submission_id>          # same rule; leaks only a status string
